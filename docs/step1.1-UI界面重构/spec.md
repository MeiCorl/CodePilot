# Step 1.1 — UI 交互界面重构

## 背景

Step 1 已完成基于 Bubble Tea 的 TUI 交互界面。在实际使用中发现 TUI 在以下方面体验欠佳：

- **视觉表现力受限**：终端字符渲染难以呈现丰富的版式、动画、代码高亮、Markdown 样式细节
- **多面板布局受限**：TUI 难以同时展示会话历史、当前会话、状态信息等多区域并存
- **交互元素受限**：无法呈现鼠标悬停、下拉候选、弹窗确认等现代 UI 元素
- **截图与协作不便**：终端界面难以嵌入文档、博客或截图分享

本步骤将 TUI 完全替换为 WebUI 页面（HTTP + WebSocket），在浏览器中提供现代化的交互体验。Agent 的业务能力（流式对话、会话持久化、上下文管理、命令解析、错误处理）保持不变，仅替换交互层。

## 目标用户

使用桌面或笔记本的开发者，在本地终端启动 `codepilot` 后，系统自动打开浏览器进入 Web 交互页面，用户在浏览器中完成所有交互（输入、查看历史、切换会话、触发命令等）。

## 能力清单

1. **Web 服务启动**：执行 `codepilot` 命令后，后端启动 HTTP 服务监听 `http://localhost:8969`，并自动调用系统默认浏览器打开交互页面
2. **静态资源托管**：后端通过 `embed.FS` 嵌入前端 HTML/CSS/JS 资源，浏览器加载后无需额外网络请求
3. **WebSocket 全双工通信**：浏览器与 Agent 通过单一 WebSocket 连接完成所有实时数据交换（用户输入、流式输出、状态推送、会话列表等）
4. **自动打开浏览器**：跨平台调用系统命令打开默认浏览器（Windows / macOS / Linux），打开失败时仅在终端打印提示，不阻塞服务
5. **顶部信息栏**：常驻展示 CodePilot LOGO、产品名称、版本号、项目开源地址、当前工作空间根目录
6. **左侧会话历史栏**：列出本地所有历史会话（按更新时间倒序），每条展示更新时间、消息数、首条用户消息预览；点击切换到该会话并加载其完整历史
7. **中间当前会话栏**：全屏最大区域，渲染当前会话的完整消息历史，用户消息与助手消息左右分列；助手消息以 Markdown 渲染（含代码高亮、列表、引用等）
8. **底部输入栏**：多行 textarea，支持 Enter 发送、Shift+Enter 换行；右侧状态区实时展示当前模型名称、上下文窗口剩余百分比、Agent 状态（空闲 / 思考中 / 错误）
9. `**/` 快捷命令下拉候选**：输入框中输入 `/` 时弹出下拉候选列表，包含本步骤已实现的 `/new`、`/sessions`、`/resume <id>` 等命令；支持键盘上下选择 + Enter 补全
10. **流式响应渲染**：助手消息以打字机效果逐字渲染，过程中显示闪烁光标；中断或完成后转为静态文本
11. **中断流式响应**：用户可通过界面按钮（停止按钮）中断当前流式响应，已输出部分保留为一条完整助手消息
12. **多会话切换**：在左侧会话栏点击目标会话后，中间区域重渲染为目标会话的完整历史；切换过程中保留当前会话的最新编辑状态
13. **新建会话**：通过 `/new` 命令或顶部按钮创建新会话，会话内容写入本地 JSON 文件
14. **API 错误友好提示**：LLM 调用失败时在中间会话区域底部以红色卡片展示错误信息，状态栏切换为"错误"状态，用户可继续输入或新建会话
15. **优雅退出**：关闭浏览器标签后后端继续运行；用户通过终端 Ctrl+C 中断后端时，自动保存当前会话到磁盘后退出
16. **完全删除 TUI 代码与依赖**：本步骤完成后，工程中不再存在 Bubble Tea、Lipgloss、Glamour、Bubbles 等 TUI 框架的引用与依赖

## 非功能要求

1. **性能**：流式响应的首 token 延迟不因 Web 改造而增加；前端单帧重绘开销不阻塞 UI 响应（流式过程中页面滚动、按钮点击、输入框编辑均流畅）
2. **可用性**：浏览器自动打开失败时仅打印提示不阻塞服务；WebSocket 断连时前端展示重连提示；LLM 错误不导致进程退出
3. **安全性**：HTTP 服务仅绑定 `127.0.0.1`，不暴露到局域网或公网；配置文件中的 API Key 不在浏览器侧代码中明文暴露（仅存在于后端配置加载时）
4. **兼容性**：支持 Chrome / Firefox / Edge / Safari 等现代浏览器最新版（不要求兼容 IE）
5. **可扩展性**：WebSocket 消息协议采用 JSON 格式，预留扩展字段；前端组件按区域拆分（top bar / session list / chat panel / input bar / status bar），新增区域不需重构现有结构
6. **UI 视觉**：采用深色编辑式美学（参考 Linear / Vercel / Raycast 风格的现代极简风），使用独特的 display 字体 + 编辑器风格 monospace 字体，配合精准分割线与微妙阴影；严禁使用 Inter / Roboto / Arial 等通用 AI 字体，严禁紫色渐变白底等 AI slop 套路
7. **可维护性**：前端零构建步骤，HTML/CSS/JS 直接以源码形式 embed 到 Go 二进制；修改前端代码后重新 `go build` 即可
8. **跨平台**：自动打开浏览器需在 Windows、macOS、Linux 三平台都能工作；端口冲突时打印明确错误并退出

## 设计骨架

```
codepilot/
├── src/
│   ├── main.go                     # 启动入口：加载配置 → 启动 Web 服务 → 打开浏览器
│   ├── internal/
│   │   ├── interaction/            # 第 1 层：交互层（重写）
│   │   │   └── web/                # Web 交互实现（原 tui 目录被删除）
│   │   │       ├── server.go       # HTTP 服务 + WebSocket 升级
│   │   │       ├── websocket.go    # WebSocket 连接管理与消息路由
│   │   │       ├── handler.go      # 业务消息处理（用户输入 / 流式 chunk / 会话命令）
│   │   │       ├── static/         # 前端静态资源（被 embed）
│   │   │       │   ├── index.html  # 单页应用入口
│   │   │       │   ├── style.css   # 全局样式（深色编辑式美学）
│   │   │       │   ├── app.js      # 前端逻辑（WebSocket / DOM / Markdown 渲染）
│   │   │       │   └── vendor/     # 第三方库（marked.js 等，源码嵌入）
│   │   │       └── browser.go      # 跨平台自动打开浏览器
│   │   ├── engine/                 # 第 2 层：引擎层（不变）
│   │   │   └── conversation/       # 对话管理
│   │   ├── memory/                 # 第 4 层：记忆层（不变）
│   │   │   ├── context/            # 滑动窗口
│   │   │   └── session/            # 会话持久化
│   │   ├── tool/                   # 第 3 层：工具层（不变）
│   │   ├── config/                 # 配置加载（不变）
│   │   └── logger/                 # 日志系统（不变）
│   └── llm/                        # LLM 适配器（不变）
├── config/
│   └── config.example.json         # 配置文件示例（不变）
├── docs/
│   └── step1.1-UI界面重构/         # 本步骤设计文档
└── go.mod / go.sum                 # 依赖：移除 Bubble Tea 系，引入 gorilla/websocket 等
```

### WebSocket 消息协议

- 客户端 → 服务端：`user_input`、`new_session`、`resume_session`、`list_sessions`、`abort_stream`
- 服务端 → 客户端：`stream_chunk`、`stream_done`、`stream_error`、`session_list`、`session_loaded`、`status_update`、`context_usage`
