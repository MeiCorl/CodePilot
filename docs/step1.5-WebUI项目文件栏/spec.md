# Step 1.5 - WebUI 项目文件栏

## 背景

CodePilot 的 WebUI 已经能展示会话、工具调用、富文本回答和文件 diff，但用户在浏览当前项目结构时仍需要切回 IDE 或终端。对于以项目为中心的 AI Coding Agent，用户应当能在同一个 WebUI 中快速查看当前启动路径下的目录、进入子目录，并打开文本文件预览内容，从而减少上下文切换。

本需求属于第 1 层交互层的 WebUI 增强，沿用现有 HTTP 静态资源 + WebSocket 业务消息架构，不改变 Agent Loop、工具系统或会话持久化协议。

## 目标用户

- 使用 CodePilot WebUI 进行代码阅读、修改和验证的开发者。
- 希望在对话旁边快速查看项目结构、定位文件并查看内容的用户。
- 需要查看 Markdown、JSON、XML、代码等文本资源，但不希望打开外部编辑器的审阅者。

## 能力清单

1. WebUI 右侧新增项目文件栏，根目录为 CodePilot 启动时的工作目录。
2. 文件栏展示当前目录下的子目录和文件，目录在前、文件在后，并按名称稳定排序。
3. 目录项可点击进入子目录；支持返回上级目录和面包屑导航。
4. 文件项可点击打开弹窗预览文件内容。
5. Markdown 文件在弹窗中以 Markdown 渲染展示，并复用现有 XSS 过滤策略。
6. JSON 文件在弹窗中进行格式化展示并使用代码高亮；格式异常时按纯文本代码展示。
7. XML、Go、JavaScript、TypeScript、CSS、HTML、YAML、Shell、Python、SQL 等文本/代码文件按语言高亮展示。
8. 普通文本文件按纯文本预览展示。
9. 二进制文件不展示内容，弹窗中给出不可预览提示。
10. 文件内容读取限制在项目根目录内，禁止通过 `..`、绝对路径伪造、符号链接逃逸访问项目外文件。
11. 大文件不直接回传完整内容，给出文件过大提示。
12. 文件栏刷新、进入目录和打开文件时有清晰的加载、空目录、错误状态。

## 非功能需求

- **安全性**：所有前端传入路径必须按项目根目录解析和校验；符号链接最终路径若逃逸项目根目录，拒绝读取或进入。
- **性能**：目录列表只读取当前层级，不递归扫描；单次列表结果设置合理上限，避免进入超大目录导致 UI 卡顿。
- **稳定性**：新增协议不影响既有 `tool_call_start` / `tool_call_end` / `session_loaded` 等消息。
- **兼容性**：不引入前端构建工具；继续使用原生 `app.js`、`style.css`、`index.html` 和现有 vendor 库。
- **可维护性**：文件浏览后端逻辑集中在 Web 交互层，作为 UI 只读能力，不复用 LLM 工具权限链路，也不写入 session JSON。

## 设计骨架

```text
src/internal/interaction/web/
├── project_file.go              # 新增：项目文件列表/读取、路径校验、类型识别
├── project_file_test.go         # 新增：路径安全、文本/二进制识别、目录列表单测
├── protocol.go                  # 修改：新增 list_project_dir/read_project_file 协议 payload
├── handler.go                   # 修改：注册并处理项目目录列表和文件读取消息
└── static/
    ├── index.html               # 修改：新增右侧文件栏与文件预览弹窗容器
    ├── style.css                # 修改：三栏布局、文件栏、预览弹窗、响应式样式
    └── app.js                   # 修改：文件栏状态、WS 请求、目录导航、文件渲染
```

核心交互链路：

```text
WebUI 打开/会话加载
  -> 前端请求 list_project_dir(path="")
  -> 后端在 workdir 内列出当前层级
  -> 前端渲染右侧文件栏

点击目录
  -> 前端请求 list_project_dir(path="相对目录")
  -> 后端校验目录仍在 workdir 内
  -> 前端更新列表与面包屑

点击文件
  -> 前端请求 read_project_file(path="相对文件")
  -> 后端校验路径、检测二进制和大小、读取文本
  -> 前端按 markdown/json/xml/code/plain 渲染弹窗
```

## Out of Scope

- 不支持编辑、删除、重命名、上传、拖拽移动等写操作。
- 不递归构建完整项目树，不做全文搜索。
- 不展示二进制文件内容，不做图片/PDF/Office 预览。
- 不把文件预览内容写入会话历史或记忆系统。
- 不替代 `ReadFile` 工具，也不改变 LLM 工具可访问范围。
- 不新增前端框架、打包器或在线 CDN 依赖。
