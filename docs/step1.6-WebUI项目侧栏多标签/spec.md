# Step 1.6 — WebUI 项目侧栏多标签

## 背景

当前 WebUI 右侧项目侧栏只有文件列表能力，适合浏览与预览项目文件，但缺少开发者日常最常用的两个入口：查看当前项目改动、在项目内检索内容。用户需要像 VSCode 或 Cursor 一样，在同一个侧栏区域中快速切换「文件」「Git」「搜索」视图，并能从 Git 变更或搜索结果直接进入可读内容。

本步骤在 Step 1.5 项目文件栏基础上扩展项目侧栏，不替换现有会话区、主对话区、工具 diff 弹窗和权限确认流程。

## 目标用户

- 使用 CodePilot WebUI 进行本地项目开发的用户。
- 需要快速查看当前工作区改动文件、比较变更内容的用户。
- 需要在项目内按关键词、路径、正则和排除规则搜索代码或文档的用户。

## 能力清单

1. 项目侧栏从单一文件列表升级为多标签结构，包含文件、Git、搜索三个入口。
2. 文件标签保留现有目录浏览、面包屑导航、刷新和文件预览能力。
3. Git 标签展示当前项目下有变更的文件，区分常见变更状态，并提供刷新、加载中、空状态和错误状态。
4. 在 Git 标签点击变更文件后，弹出或打开左右双栏变更内容视图，展示变更前与变更后的文件内容。
5. 搜索标签支持输入关键词并返回包含关键词的文件列表与命中片段。
6. 搜索标签支持限定搜索路径、启用正则匹配、配置排除文件或目录规则。
7. 搜索与 Git 能力均遵循项目工作目录边界，不读取或展示项目根之外的内容。
8. 后端对大文件、二进制文件、过多结果和外部命令不可用等情况提供稳定错误原因，前端以可理解状态展示。

## 非功能要求

- 右侧侧栏布局应保持当前 WebUI 风格，标签切换不能挤压主对话区和输入区。
- Git 与搜索请求应限制扫描范围、文件大小和结果数量，避免卡住 UI 或阻塞主流程。
- 所有新增 WebSocket 协议应保持结构化、可扩展，和现有 `project_dir` / `project_file` 交互风格一致。
- 前端需要过滤过期回包，避免快速切换标签、重复搜索或刷新时显示旧结果。
- Git 命令不可用、当前目录不是 Git 仓库、搜索输入非法等情况不能导致服务端 panic 或前端崩溃。

## 设计骨架

```text
src/internal/interaction/web/
  project_file.go          # 复用项目路径边界、文本识别和文件预览基础能力
  project_git.go           # 新增 Git 变更列表与双栏内容读取
  project_search.go        # 新增项目内容搜索模型与扫描逻辑
  protocol.go              # 扩展项目侧栏相关 WS 消息与 payload
  handler.go               # 注册并处理 Git / 搜索请求
  *_test.go                # 补充 Git / 搜索 / 协议级测试

src/internal/interaction/web/static/
  index.html               # 项目侧栏多 tab DOM
  style.css                # tab、Git 列表、搜索表单与结果样式
  app.js                   # 侧栏 tab 状态、Git 请求、搜索请求、diff 展示

docs/step1.6-WebUI项目侧栏多标签/
  spec.md
  tasks.md
  checklist.md
```

## Out of Scope

- 不实现提交、暂存、撤销、分支切换、解决冲突等 Git 写操作。
- 不实现全仓库实时索引、后台增量索引或跨项目搜索。
- 不引入复杂搜索语法、结构化 AST 搜索或语义搜索。
- 不把 Git 变更与工具调用产生的 `file_diff` 记录混为一个数据源；二者可以复用展示组件，但数据来源保持独立。
- 不改变现有 Agent Loop、工具系统、权限系统和会话持久化协议。
