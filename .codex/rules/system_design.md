# Agent架构与代码生成强制规范

## 一、架构设计强制要求
1. 架构必须覆盖**高性能、高可用、高扩展、高安全**四大基础原则
    - 高性能：规避无效计算、重复请求，合理使用缓存/异步/批量/限流；减少冗余链路
    - 高可用：规避单点故障，增加异常捕获、重试、降级、故障兜底机制
    - 高扩展：核心与扩展逻辑解耦，业务参数可配置，避免硬编码，新增功能不改动核心架构
    - 高安全：入参校验脱敏、敏感信息禁止明文、接口权限控制、防篡改越权
2. 复杂系统必须分层/分模块设计，简单工具系统可简化
    - 每层/模块单一职责，做到高内聚、松耦合
    - 模块边界清晰，仅通过标准接口交互，禁止依赖内部私有逻辑
    - 输出产物：模块结构图、各模块职责、模块依赖关系、扩展约束

## 二、代码模块划分规则
1. 代码包/目录结构必须与架构分层、模块一一对应，不同职责代码禁止混放
2. 同类实体、逻辑、工具收敛至同一模块，单个类/方法仅承担单一功能
3. 新增代码遵循原有目录结构，禁止随意创建临时目录、冗余模块

## 三、代码注释强制规范
### 1. 必须写注释对象
- 核心类/结构体：说明功能、业务场景、对外能力
- 核心函数/方法：描述功能、入参含义&约束、返回值、特殊逻辑
- 核心成员变量/参数：说明用途、取值范围、默认值、业务状态
- 复杂逻辑关键步骤：说明该段代码设计目的与处理逻辑

### 2. 注释通用约束
- 使用简体中文，描述准确无歧义
- 注释和代码逻辑保持同步，禁止无效、过期注释

## 四、关键代码路径增加适当日志，便于问题排查

## 五、输出自检清单（生成完成必须自查）
- [ ] 架构满足高性能、高可用、高扩展、高安全四项要求
- [ ] 复杂系统完成分层分模块，高内聚松耦合，职责边界清晰
- [ ] 代码目录结构匹配架构模块划分
- [ ] 核心类、方法、字段、关键代码步骤全部配备有效注释
- [ ] 不存在架构冗余、模块强耦合、代码堆砌、注释缺失问题
