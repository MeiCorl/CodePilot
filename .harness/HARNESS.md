# 背景

我正在从零开始构建一个终端AI Coding Agent（类似 Claude Code），项目名叫CodePilot，使用的编程语言是Go语言。最终希望实现CodePolit以命令行界面启动，根据用户输入、通过Agent Loop循环与LLM交互并调用工具，最终实现用户要求，当然也需要实现上下文管理、记忆系统、MCP协议、Skill系统、工具系统、SubAgent等一系列Agent常用功能及组件。具体实现步骤计划如下：

- step1: LLM打通
- step2: 工具系统集成
- step3: ReAct与Agent Loop实现
- step4: System Promt设计
- step5：权限系统设计
- step6: MCP协议实现
- step7: 上下文管理
- step8: 记忆系统
- step9: 快捷命令系统
- step10：Skill系统
- step11：Hook系统
- step12: SubAgent

其中每一步我们都将使用「spec工作流」对功能点进行深入、细化拆分，相关分析结果以及执行计划将放到docs/{步骤名}目录下，可以从该目录下分析出当前功能步骤进行到哪了，下一次该从哪继续上次未完成的工作；也可以从docs目录下看出当前整个系统设计实现进行到哪一步。

# 系统架构

CodePilot 采用 **5 层垂直分层架构**，代码目录结构必须与架构分层一一对应。每层职责单一、高内聚，层间仅通过标准接口交互，禁止跨层直接依赖内部实现。

## 第 1 层：交互层（Interaction Layer）

用户入口与交互方式管理，决定用户如何与 Agent 进行沟通。

- **WebUI**： Web交互页面，Agent 的主入口。负责终端 UI 渲染、输入捕获、输出格式化（Markdown 渲染、代码高亮等）。
- **命令**：快捷命令系统（`/help`、`/clear`、`/init` 等斜杠命令），将用户意图映射为特定操作，绕过 Agent Loop 直接执行。
- **Skill**：技能系统，可插拔的能力模块。将复杂工作流封装为可复用的 Skill（如代码审查、测试生成），用户通过 `/skill-name` 触发。

## 第 2 层：引擎层（Engine Layer）

Agent 的核心驱动，实现 ReAct 推理循环与 LLM 交互编排。

- **对话**：对话管理，维护与 LLM 的多轮消息流，处理消息的构造、发送、响应解析（包括 tool_use 等结构化输出）。
- **循环**：Agent Loop 的核心引擎。实现「思考→决策→行动→观察」的迭代循环，决定何时继续执行、何时终止、何时切换策略。
- **提示词**：System Prompt 管理。负责系统提示词的组装、模板渲染、动态注入（如项目上下文、用户偏好、权限状态等）。

## 第 3 层：工具层（Tool Layer）

Agent 的"双手"，提供与外部世界交互的能力。

- **工具**：内置工具集（文件读写、代码搜索、Shell 执行等），每个工具定义明确的输入 Schema、执行逻辑和输出格式。
- **MCP**：Model Context Protocol 实现，通过标准协议接入外部工具服务器，实现工具的动态发现与调用，扩展 Agent 能力边界。
- **Hook**：钩子系统，在工具执行前后插入自定义逻辑（如日志记录、权限拦截、结果过滤），实现事件驱动的扩展机制。
- **SubAgent**：子代理系统，由主 Agent 派生出独立子代理执行子任务，支持并行调度、上下文隔离与结果回传，用于处理复杂的多步骤工作。

## 第 4 层：记忆层（Memory Layer）

状态持久化与上下文管理，让 Agent 具备"记忆力"。

- **上下文**：上下文管理，负责对话窗口的构建、压缩（摘要）、缓存，确保在 LLM 上下文窗口有限的情况下保留最相关信息。
- **会话**：会话管理，负责会话的创建、恢复、持久化，支持用户中断后恢复工作、多会话并行等场景。
- **自动记忆**：自动记忆系统，Agent 自主判断哪些信息值得长期记住（如用户偏好、项目约定），跨会话持久化并在适当时机召回。

## 第 5 层：安全层（Security Layer）

横切关注点，为所有上层操作提供安全保障。

- **权限**：权限控制系统，定义工具调用的白名单/黑名单、危险操作确认机制（如文件删除、命令执行需用户批准）。
- **沙箱**：沙箱隔离，为高风险操作（如 Shell 命令执行、文件写入）提供隔离执行环境，防止对宿主系统造成不可逆损害。
- **HITL**：Human-in-the-Loop 人工干预机制，在关键决策节点暂停自动执行，请求人类确认或引导，确保 Agent 行为始终在人类监督之下。

## 架构约束

1. **依赖方向**：上层可调用下层接口，下层禁止依赖上层。安全层作为横切层可被任意层调用。
2. **模块划分**：每层的组件对应独立的代码包，不同职责代码禁止混放。
3. **接口交互**：层间仅通过标准接口交互，禁止依赖其他层/模块的内部私有逻辑。
4. **SubAgent**：归属工具层，作为主 Agent 可调用的特殊"工具"，由引擎层编排调度。

# 其它通用规定

1. 使用 SDK 时务必通过 context7 MCP 等文档工具查最新API（函数签名、参数顺序、类型名），不要凭记忆写。
2. 修改或新增功能时一定要有全局意识，明确CodePilot最终需要实现的计划和目的，确保本次修改/新增既要兼容已有功能设计，也要为后续功能的开发接入提供便捷性
3. 所有回复输出默认使用中文
